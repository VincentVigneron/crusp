fn box_clone(&self) -> Box<Constraint<Handler>>
[src]

Constraints have to define box_clone in order to be cloned. The ConstraintsHandler handle many kind of variables, so it uses constraints as trait object. Trait object can not be cloned because Clone require the trait to be Sized but trait object can not be Sized. box_clone is a way to bypass this requirement, but box_clone can not guarantee that the clone is the same as the original, it is the Constraint developper to guarantee that the clone is equivalent to the original. Initialisation should guarantee that the Constraint does not manipulate twice the same view. Since view of ArrayOfRefs does not hold any information about their underlying views (variables), it is necessary to ask this information to the variables handler. Indeed, VariableView has to implement the Copy trait. The structures implementing Copy must support a memcopy (i.e. the structure can be copy byte by byte and its size must be known at compile time), nevertheless ArrayOfRefs refers to non-contigous variables in memory so, in general, it requires to hold a dynamic array (Vec) of variables which the size is known at runtime.
